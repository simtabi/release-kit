# AWS Elastic Container Registry (ECR)

**Automation level**: CLI login + API
**Source-of-truth**: `Dockerfile`

## Overview

ECR comes in two flavours:

- **Private ECR** — per-AWS-account, per-region. Pay per GB.
  Reference: `<account>.dkr.ecr.<region>.amazonaws.com/<repo>:<tag>`.
- **Public ECR** (`public.ecr.aws`) — free for public images.
  Reference: `public.ecr.aws/<alias>/<repo>:<tag>`. Available only
  in `us-east-1`.

Both share the same auth flow (a short-lived password from
`get-login-password`).

## Account & project bootstrap

1. AWS account exists.
2. Decide private vs public.
3. **Private ECR**: console → ECR → Create repository → enter
   name, immutability (recommended), encryption (KMS or AES-256).
4. **Public ECR**: console → ECR Public → Create repository →
   pick a public alias (one-time, irreversible).
5. (Optional) Set repo policy + lifecycle policy at creation.

## Authentication options (ranked: most secure → least)

1. **OIDC via GitHub Actions / GitLab CI** through
   `aws-actions/configure-aws-credentials` (GitHub) or
   `id_tokens:` (GitLab) — short-lived federated credentials.
2. **IAM role** assumed by a CI runner (e.g., EC2 instance profile,
   EKS service account).
3. **IAM user access key + secret** — long-lived static keys.
   Rotate every 90 days.

ECR never accepts username/password directly; the password is
always generated by `aws ecr get-login-password`.

## One-time setup

### OIDC trust policy (AWS IAM)

```bash
# bash
# 1. Create an OIDC identity provider for GitHub
aws iam create-open-id-connect-provider \
  --url https://token.actions.githubusercontent.com \
  --client-id-list sts.amazonaws.com \
  --thumbprint-list <github-cert-thumbprint>

# 2. Trust policy: allow this repo's workflow to assume the role
cat > trust.json <<'EOF'
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": { "Federated": "arn:aws:iam::<account>:oidc-provider/token.actions.githubusercontent.com" },
    "Action": "sts:AssumeRoleWithWebIdentity",
    "Condition": {
      "StringEquals": {
        "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
      },
      "StringLike": {
        "token.actions.githubusercontent.com:sub": "repo:simtabi/release-kit:*"
      }
    }
  }]
}
EOF
aws iam create-role --role-name release-kit-ecr-push \
  --assume-role-policy-document file://trust.json

# 3. Attach ECR push policy (see ../cross-cutting/token-scoping.md)
aws iam attach-role-policy --role-name release-kit-ecr-push \
  --policy-arn arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryPowerUser
# Or a custom policy with just the per-repo actions.
```

### GitHub Actions workflow

```yaml
# bash / yaml
publish-ecr:
  runs-on: ubuntu-latest
  permissions:
    id-token: write
    contents: read
  steps:
    - uses: actions/checkout@v4
    - uses: aws-actions/configure-aws-credentials@v4
      with:
        role-to-assume: arn:aws:iam::<account>:role/release-kit-ecr-push
        aws-region: us-east-1
    - uses: aws-actions/amazon-ecr-login@v2
      id: ecr
      with:
        registry-type: private    # or "public"
    - uses: docker/setup-buildx-action@v3
    - uses: docker/build-push-action@v6
      with:
        push: true
        platforms: linux/amd64,linux/arm64
        tags: |
          ${{ steps.ecr.outputs.registry }}/release-kit:${{ github.ref_name }}
          ${{ steps.ecr.outputs.registry }}/release-kit:latest
```

### Manual / local

```bash
# bash
aws ecr get-login-password --region us-east-1 \
  | docker login --username AWS --password-stdin <account>.dkr.ecr.us-east-1.amazonaws.com

# Public ECR:
aws ecr-public get-login-password --region us-east-1 \
  | docker login --username AWS --password-stdin public.ecr.aws
```

## Per-release workflow

### Manual

```bash
# bash
ACCT=<account>
REGION=us-east-1
REPO=release-kit
TAG=1.4.2

docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag $ACCT.dkr.ecr.$REGION.amazonaws.com/$REPO:$TAG \
  --push .
```

### CI/CD

See workflow above.

## Verification

```bash
# bash
aws ecr describe-images --repository-name release-kit --image-ids imageTag=1.4.2
docker pull <account>.dkr.ecr.us-east-1.amazonaws.com/release-kit:1.4.2
```

## Common failure modes & fixes

| Symptom | Cause | Fix |
|---|---|---|
| `denied: User: arn:aws:sts::... is not authorized to perform: ecr:InitiateLayerUpload` | IAM policy missing | See [`../cross-cutting/token-scoping.md::AWS ECR`](../cross-cutting/token-scoping.md) |
| `RepositoryNotFoundException` | Repo doesn't exist | `aws ecr create-repository --repository-name release-kit` |
| `403 Forbidden` from Docker after `aws ecr login` | Password rotated; cached creds stale | Re-run `get-login-password \| docker login` |
| `manifest invalid` for multi-arch | Repo image-format mismatch | Recreate repo with default OCI manifest support |
| Layers slow to upload | Region mismatch (e.g., pushing from EU to us-east-1) | Use a closer region |

## Security checklist

- [ ] Repo created with **image tag immutability** = ON (prevents
      tag overwrites).
- [ ] Repo created with **scan on push** = ON (basic CVE scan).
- [ ] Lifecycle policy in place (delete untagged + old tags).
- [ ] Encryption at rest set (AES-256 default; KMS for stricter).
- [ ] IAM role / policy scoped to specific repos, not `*`.
- [ ] No long-lived access keys; prefer OIDC or role chaining.
- [ ] CloudTrail enabled to audit ECR pushes.
